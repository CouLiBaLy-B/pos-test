# Feature Development

Quand une nouvelle fonctionnalité est demandée :

1. reformuler brièvement l'objectif
2. repérer les fichiers touchés
3. proposer un plan en petites étapes
4. implémenter par incréments simples
5. ajouter ou mettre à jour les tests
6. vérifier les impacts de bord
7. résumer les changements

## Règles

- éviter les refactors larges non demandés
- préférer des changements localisés
- vérifier les conventions déjà présentes dans le repo
- expliquer les compromis si plusieurs options existent
