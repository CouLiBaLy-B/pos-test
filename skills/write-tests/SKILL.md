# Write Tests

Objectif : écrire des tests utiles, petits, lisibles et centrés sur le comportement.

## Processus

1. identifier les comportements critiques
2. repérer les cas nominaux
3. repérer les cas limites
4. écrire les tests minimums qui protègent la fonctionnalité
5. éviter les mocks inutiles
6. ne pas sur-spécifier l'implémentation
