# Git Workflow

## Avant commit

1. lister les fichiers modifiés
2. vérifier qu'aucun secret n'est inclus
3. retirer les artefacts temporaires
4. résumer les changements
5. proposer un message de commit clair

## Convention conseillée

- `feat:` nouvelle fonctionnalité
- `fix:` correction
- `docs:` documentation
- `refactor:` refactor sans changement fonctionnel
- `test:` ajout ou mise à jour de tests
- `chore:` maintenance
